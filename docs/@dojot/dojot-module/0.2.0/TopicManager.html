Methods

(async) getTopic(subject, tenant, broker, global)

Retrieve a topic from Data Broker.

This function will request Data Broker for a Kafka topic, given a subject and a tenant. It will retry a few times (configured in config object) and will reject the promise if it is unable to reach Data Broker for any reason. This promise will also be rejected if the response is not a valid JSON or contains invalid data.

All topics are cached in order to speed up future requests.

If true, the global parameter will indicate to Data Broker that the requested topic should be the same for all tenants. This is useful when dealing with topics that are intended to contain messages not related to tenants, such as messages related to services.

Parameters:
Name Type Description
subject string

The subject related to the requested topic
tenant string

The tenant related to the requested topic
broker string

URL where data broker can be accessed
global boolean

true if this topic should be sensitive to tenants
